# 记忆管理 SOP (L0)

参考 [pc-agent-loop](https://github.com/lsdefine/pc-agent-loop) 的记忆宪法，简化适配 GUI 自动化场景。

## 核心原则

1. **行动验证原则**：只有成功执行完成的任务才可保存为 SOP。失败或中断的不保存。
2. **语义总结**：步骤由 LLM 总结为可读的语义描述（如「点击 Dock 栏的 WPS 图标」），便于召回时理解与复用。
3. **可复用优先**：优先保存跨会话可复用的操作流程（如「打开某网站」「搜索某类内容」）。

## 记忆层级

- **L2** `global_facts.json`：环境事实（路径、配置等）
- **L3** `sops/*.json`：任务 SOP，自主学习增长

## SOP 更新规则

- 新任务完成 → 若与已有 SOP 相似则增加 success_count，否则新建
- 仅保存验证成功的步骤序列
- 禁止存储：密码、API Key、具体像素坐标（使用相对坐标）
